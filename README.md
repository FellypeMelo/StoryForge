O **StoryForge** não é apenas um editor de texto; é um laboratório de engenharia narrativa operando sob o paradigma **AI-XP (Artificially Intelligent eXtreme Programming)**. A plataforma orquestra um esquadrão de agentes especializados projetados para combater o *LLM Bloat*, abolir o *vibe coding* literário e elevar a ficção humana através de metodologias narrativas de classe mundial.

> **Filosofia Central:** A IA gera **30% da tração bruta**. Os outros **70% são modelagem fina humana** — a dor, a sujeira, as idiossincrasias que nenhum modelo estocástico pode replicar.

---

## 📐 Princípios Arquiteturais

| Princípio | Descrição |
|-----------|-----------|
| **Offline-First Absoluto** | Dados nunca saem da máquina sem consentimento explícito. SQLite local como única fonte de verdade. |
| **Anti-Averaging Effect** | Cada agente possui guardrails negativos que bloqueiam a saída estatisticamente mais provável (clichês, purple prose, diálogos de RH). |
| **Causalidade Inquebrável** | Eventos conectados por "portanto" ou "mas" — nunca por "e então". Toda cena deve inverter polaridade de valor (Story Grid). |
| **Economia de Contexto** | Injeção RAG cirúrgica: apenas trechos relevantes da Bíblia da História entram no prompt. Zero inundação de tokens. |
| **Regra 70/30** | 30% IA + 70% humano. A Fase 6 (Polimento Literário) é **100% humana** — intransferível. |

## 🔄 Workflow Profissional (Pipeline de 6 Fases)

O StoryForge implementa um pipeline completo de produção literária onde cada fase alimenta a próxima:

```
┌─────────────────────────────────────────────────────────────────────────┐
│ Fase 1: PLANEJAMENTO                                                    │
│ Ideação CHI (bloqueio de clichês) + Fichas OCEAN/Hauge                  │
│ Agentes: 01_ideation_chi + 02_character_builder                         │
└─────────────────────────────┬───────────────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ Fase 2: ARQUITETURA                                                     │
│ Beat Sheet (Cena/Sequela) + Framework Narrativo + Story Grid            │
│ Agentes: 03_narrative_structure + 04_chapter_architect                   │
└─────────────────────────────┬───────────────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ Fase 3: ESCRITA GUIADA (E.P.R.L.)                                       │
│ Prosa de 500-800 palavras por beat (CAD + Guardrails + RSIP)            │
│ Agente: 05_writing_assistant | Consulta: 06_story_bible (RAG)           │
└─────────────────────────────┬───────────────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ Fase 4: REFINAMENTO (Multi-Perspectiva)                                 │
│ 3 variações (cínica, sensorial, poética) → Fusão Frankenstein           │
│ Agente: 08_advanced_techniques (MPS)                                    │
└─────────────────────────────┬───────────────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ Fase 5: EDIÇÃO ESTRUTURAL                                               │
│ Auditoria developmental: causação, agência, show/don't tell, IA-ismos  │
│ Agente: 07_bad_cop_editor (detectores heurísticos em background)        │
└─────────────────────────────┬───────────────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ Fase 6: POLIMENTO LITERÁRIO (100% HUMANO)                               │
│ Ajuste fino de cadência, corte de adjetivos, garantia da "Bossa"        │
│ Agentes: NENHUM — esta fase é intransferível.                           │
└─────────────────────────────────────────────────────────────────────────┘
```

## 🏗️ Arquitetura Tecnológica (Stack)

> **Design Principle:** Offline-first absoluto, privacidade criptográfica e arquitetura tolerante a falhas baseada em IPC puro (Inter-Process Communication).

| Camada | Tecnologia | Função |
|--------|-----------|--------|
| **Core & Runtime** | Rust (Tauri) | Backend de sistema com IPC, segurança nativa e binário portável |
| **Frontend / UI** | React + Tailwind CSS | Interface de escrita livre de distrações, design modular |
| **Persistência** | SQLite + `sqlite-vec` + FTS5 | Motor relacional local com busca vetorial semântica para RAG |
| **Orquestração GenAI** | API unificada multi-provedor | OpenAI, Anthropic Claude, Google Gemini, Ollama (Llama 3 local) |
| **Orquestração de Agentes** | CrewAI (local) + LangGraph (CI/CD) | Squads modulares + pipelines ciclomáticos de infraestrutura |
| **CI/CD & Segurança** | LangGraph + SonarQube (SAST) | Pipeline automatizado com remediação agêntica self-healing |

## 🧲 Diferenciais Técnicos

### Motor RAG da Bíblia da História

A **Bíblia da História** é o coração do sistema de consistência narrativa. Funciona como um Lorebook inteligente com pipeline RAG:

| Seção | Conteúdo |
|-------|----------|
| **Personagens** | Fichas OCEAN + Hauge + Voz + Tells físicos |
| **Locais** | Descrição física, atmosfera, significado simbólico |
| **Regras do Mundo** | Física/magia → Economia → Sociologia (hierarquia CAD) |
| **Cronologia** | Timeline com dependências causais |
| **Relacionamentos** | Grafos de relação entre personagens |
| **Blacklists** | Clichês proibidos, palavras banidas, IA-ismos |
| **Estado Atual** | Snapshot de cada personagem no ponto atual da narrativa |

**Alertas automáticos** de inconsistência (factual, temporal, caracterização, regras do mundo) rodam em background e sinalizam contradições antes que contaminem a narrativa.

### Detectores Heurísticos (Bad Cop Editor)

O módulo de auditoria automatizada inclui:

- **Blacklist de AI-ismos** — Busca por termos como "rica tapeçaria", "mosaico de emoções", "dança mortal"
- **Classificador de IA** — Modelo auxiliar audita trechos em background
- **Repetição sintática** — Análise de padrões de início de parágrafos
- **Densidade de advérbios** — Contagem de "-mente" por parágrafo com threshold

### Frameworks Narrativos Suportados

| Framework | Aplicação Ideal |
|-----------|----------------|
| Jornada do Herói (Campbell/Vogler) | Fantasia, Épicos, Sci-Fi, Coming-of-Age |
| Save the Cat (Blake Snyder) | Romances comerciais, Thrillers, Young Adult |
| Curva Fichteana (John Gardner) | Mistérios, Pulp Fiction, Suspenses |
| Estrutura de 7 Pontos (Dan Wells) | Enredos complexos, planejamento reverso |
| Kishōtenketsu (4 Atos Asiáticos) | Ficção literária, Slice-of-Life |
| Narrativa Não Linear | Cronologia reversa, flashbacks temáticos |

## 🤝 Contribuindo

Este projeto opera sob o framework **AI-XP** com restrições rigorosas:

1. **TDD é mandatório** — Nenhum código de produção sem teste vermelho prévio.
2. **Clean Architecture** — O domínio nunca importa infraestrutura.
3. **YAGNI + KISS** — Proibido antecipar recursos não solicitados.
4. **Funções ≤ 15 linhas** — Classes ≤ 200 linhas.
5. **Emissão integral** — Proibido sumarizar código com `// ... código anterior aqui`.

Antes de submeter qualquer contribuição, consulte as especificações de engenharia em [`docs/engineering/`](docs/engineering/).
